# PRD (MVP)

## 프로젝트명

**figma-bridge-webapp**

## 한 줄 설명

피그마 URL을 입력하면 화면 구조 데이터와 이미지를 추출하고, 이를 바이브코딩에 바로 활용할 수 있도록 복사 가능한 형태로 제공하는 웹앱

## 1. 문제 정의

디자인을 보고 개발하거나 AI로 화면을 구현할 때, Figma의 정보를 직접 해석하는 과정이 번거롭고 시간이 오래 걸린다.
특히 Figma 원본 데이터는 복잡해서 그대로 사용하기 어렵고, 화면 이미지와 구조 정보를 따로 확인해야 하는 불편함이 있다.

## 2. 목표

사용자가 **Figma URL 하나만 입력**하면, 해당 화면의 핵심 정보를 추출해 다음 3가지를 바로 얻을 수 있게 한다.

1. 화면 이미지
2. 정리된 구조 데이터(JSON schema)
3. 바이브코딩용 복사 가능한 프롬프트/텍스트

## 3. 대상 사용자

* AI를 활용해 UI를 빠르게 개발하려는 개발자
* Figma 디자인을 개발 가능한 형태로 정리하고 싶은 기획자/디자이너
* 화면 구현 전 참고용 구조 데이터가 필요한 사용자

## 4. 핵심 사용자 흐름

1. 사용자가 웹앱에 **Figma URL** 입력
2. 시스템이 해당 화면 정보를 추출
3. 웹에서 아래 결과를 보여줌

   * 화면 이미지
   * 구조화된 JSON schema
   * 바이브코딩용 텍스트
4. 사용자가 원하는 결과를 **클릭 한 번으로 복사**
5. 복사한 내용을 AI 개발 도구나 코드 작업에 활용

## 5. MVP 핵심 기능

### 5.1 Figma URL 입력

* 사용자는 Figma 파일/프레임 URL을 입력할 수 있어야 한다.

### 5.2 화면 정보 추출

* URL에서 file key, node id를 파싱한다.
* 해당 화면의 기본 구조 데이터를 가져온다.
* 전체 화면 이미지를 추출한다.

### 5.3 구조 데이터 정리

* Figma raw 데이터 전체가 아니라, MVP에서는 다음 핵심 정보만 정리한다.

  * 화면 이름
  * 주요 텍스트
  * 주요 컴포넌트/섹션
  * 레이아웃 구조(단순화)
  * 기본 스타일 정보 일부

### 5.4 결과 표시

웹앱에서 아래 3개 결과를 보여준다.

* **화면 이미지**
* **정리된 JSON schema**
* **바이브코딩용 요약 텍스트**

### 5.5 복사 기능

* JSON 복사
* 바이브코딩용 텍스트 복사
* 복사 성공 피드백 표시

## 6. MVP 범위

### 포함

* Figma URL 입력
* URL 파싱
* 단일 화면 기준 추출
* 화면 이미지 1장 표시
* 단순화된 schema 생성
* 복사 가능한 결과 제공
* 웹 UI + FastAPI 백엔드 연동

### 제외

* 다중 화면 동시 처리
* 디자인 diff 비교
* 구현 결과 검증
* 고급 컴포넌트 매핑
* 상태별 화면 추출
* 자동 코드 생성
* 사용자 계정/팀 기능

## 7. 성공 기준

* 사용자가 URL 입력 후 결과를 확인할 수 있다.
* 결과가 30초 이내에 표시된다.
* 사용자가 JSON 또는 프롬프트를 바로 복사할 수 있다.
* 최소 1개 화면에 대해 이미지 + 구조 데이터 + 텍스트 요약이 정상 생성된다.

## 8. 기술 구성

* **Frontend:** 사용자용 웹앱
* **Backend:** FastAPI
* **External:** Figma API

## 9. 비기능 요구사항

* UI는 단순하고 직관적이어야 한다.
* 복사 동작은 최대한 빠르고 명확해야 한다.
* 추출 실패 시 에러 메시지를 사용자에게 보여줘야 한다.

## 10. 최종 정의

**figma-bridge-webapp**의 MVP는
사용자가 **Figma URL을 입력하면**, 해당 화면의 이미지와 핵심 구조 데이터를 추출해 **바이브코딩에 바로 사용할 수 있는 형태로 보여주고 복사하게 해주는 웹앱**이다.
